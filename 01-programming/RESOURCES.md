## Recommended Books

| Topic | Book | Author | Level | Why Read It |
|-------|------|--------|-------|-------------|
| Programming Craft | "Clean Code" | Robert C. Martin | Intermediate | The definitive guide to writing readable, maintainable code |
| Design Patterns | "Head First Design Patterns" | Eric Freeman & Elisabeth Robson | Intermediate | Visual, engaging approach to the Gang of Four patterns |
| Refactoring | "Refactoring: Improving the Design of Existing Code" | Martin Fowler | Intermediate | Systematic techniques for improving code without changing behavior |
| Software Design | "A Philosophy of Software Design" | John Ousterhout | Intermediate | Concise guide to reducing complexity in software systems |
| Pragmatic Approach | "The Pragmatic Programmer" | David Thomas & Andrew Hunt | Intermediate | Timeless advice on software craftsmanship and career growth |
| Python | "Fluent Python" | Luciano Ramalho | Advanced | Deep dive into Pythonic patterns and language internals |
| JavaScript | "You Don't Know JS" (series) | Kyle Simpson | Intermediate | Thorough exploration of JavaScript's core mechanisms |
| Functional Programming | "Structure and Interpretation of Computer Programs" | Abelson & Sussman | Advanced | Classic MIT textbook teaching programming through Scheme |
| Domain Modeling | "Domain-Driven Design" | Eric Evans | Advanced | How to model complex business domains in code |
